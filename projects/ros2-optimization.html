02 — DEEP DIVE

ROS2 Resource Optimization for Embodied AI

ROS2 VLA Models Edge Compute Thread Profiling Memory Optimization

The Challenge

Running complex VLA models, state machines, and continuous control loops simultaneously leads to severe CPU bottlenecking and system freezes on edge compute devices. A household cleaning robot cannot afford to overheat or lag — it needs to run advanced AI locally, in real time, without thermal throttling.

The Implementation

Conducted granular thread and memory profiling of ROS2 multithreaded executors. Restructured node communication to minimize publisher/subscriber overhead during high-frequency spine movements, resulting in a lean, highly responsive control stack capable of running advanced AI locally without thermal throttling.

Key Technical Details

  • Deep profiling of ROS2 multithreaded executor thread scheduling and CPU affinity
  • Identification of publisher/subscriber bottlenecks during high-frequency control loops (100Hz+ spine movements)
  • Node communication restructuring — reduced unnecessary message serialization and callback queue depth
  • Memory allocation optimization to prevent garbage collection spikes during manipulation tasks
  • CPU consumption reduction across complex ROS node architectures running VLA inference, state machines, and motor controllers concurrently

Outcome

Achieved a lean, responsive control stack that runs Vision-Language-Action models alongside real-time motor control on edge hardware without thermal throttling. The robot maintains consistent performance during extended cleaning sessions — no freezes, no lag, no thermal shutdowns.
